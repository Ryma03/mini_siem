1. Alert Collection (Snort Machine - Ubuntu)
Snort runs on Ubuntu machine and monitors network traffic
Detects suspicious activity based on security rules
Generates alert data in /var/log/snort/alert_fast.log
2. Alert Processing (Target Machine - Ubuntu)
Your Python SIEM application receives alerts through:

collector.py → Reads Snort logs

	Parses alert data from log file
	Extracts: source IP, destination IP, port, protocol, signature
	Normalizes to Python dictionary format
enricher.py → Enhances alert data

	Adds geolocation (country, city, coordinates)
	Adds ASN (Autonomous System Number)
	Adds organization name
	Uses IP-API.com service with caching
correlator.py → Detects attack patterns

Groups related alerts together
Detects 3 attack types:
	DoS (Denial of Service) - 5+ alerts in 10 minutes
	Reconnaissance - 3+ different signatures from same IP
	Exploitation - Rapid alerts (< 30 seconds apart)
3. Data Storage (SQLite Database)
	alerts table:
├─ id, signature, src_ip, dst_ip
├─ src_port, dst_port, protocol
├─ severity, message, timestamp
└─ enrichment_data

	correlations table:
├─ id, attack_type, src_ip
├─ alert_count, unique_signatures
└─ first_alert_time, last_alert_time
